◆ Master Plan

OmniBox — Full Consolidation & Multi-Platform Buildout

Collapse every scattered project into one monorepo, with a shared core so any fix scales to all platforms — and a cloud-phone transport (DuoPlus) that unlocks Snapchat, Instagram & Facebook.

📅 2026-06-27 🗂 6 codebases → 1 repo 🖥 2 servers 📱 DuoPlus cloud phones

01 Why we're doing this

Today the system spans ~6 codebases on 2 servers, with duplicate, drifting copies of the same platforms. Every fix has to be delivered to multiple sources and nothing is shared. The target is one repo, a shared core (fix once → every platform benefits), new platforms that only write their own automation, and cloud phones as a first-class transport.

The scaling principle

Every platform implements one PlatformTransformer contract → normalized OmniConversation/OmniMessage → unified inbox. The shared core holds accounts, proxies, warmup, scrape & inbox-sync — written once, reused everywhere. New platform = write only its automation.

06 Data strategy

During transition, each platform keeps its own store (Telegram SQLite, c2 Postgres, Discord Postgres-in-container). inbox-sync in the core normalizes everything into the existing omnibox_unified Postgres — which powers the cross-platform inbox. Full per-platform apps keep their own DBs (embedded in the shell). Consolidate further later, only once stable.

08 The phased merge

Principle: absorb source first (zero runtime disruption) → cut over runtime last → back up before every cutover. Live systems keep running throughout.
0

Foundation & safety

  • Back up all live systems (c2 DB, Discord volume, Telegram SQLite + sessions, configs)
  • Monorepo workspace tooling + CI
  • Scaffold packages/core, unified-db, transports/
  • Resolve open decisions (§09)
1

Absorb clean sources no runtime change

  • c2 → platforms/tiktok (replace stale fork; keep /root/C2 live until Phase 6)
  • AppellantBot → services/appeal-bot
  • duoapi → transports/cloud-phone + "Cloud Phones" shell section
  • Discord source → platforms/discord · Telegram consolidate
2

Extract the shared core

  • Move accounts · proxies · campaigns · warmup · scrape · inbox-sync into packages/core
  • Refactor Telegram + TikTok onto core (proves "fix once")
  • Define the device-pool abstraction (device ↔ account ↔ proxy ↔ number)
3

Harden the cloud-phone transport

  • Whitelist omnibox host IP in DuoPlus ADB
  • Appium-over-openADB harness (connect · dump UI · act)
  • RPA trigger wrapper · cloudNumber onboarding · media push
4

🎯 Snapchat pilot — validates the whole pipeline

  • Engine: install/launch → open chats → dump UI → parse → normalize → reply via Appium
  • Surface in unified inbox + shell; embed duoapi console for device ops
  • Smoke-test on 1–2 devices before fleet
5

Instagram + Facebook

  • Reuse cloud-phone transport; implement per-platform selectors/flows (engine, not infra)
6

Runtime consolidation & deploy

  • Single deploy from omnibox; cut TikTok off /root/C2; retire stale copies
  • Discord stays containerized but deployed from monorepo source
7

Unified, cross-platform features

  • Native cross-platform unified inbox
  • Cross-platform campaigns/warmup driven from the core
